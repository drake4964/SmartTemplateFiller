# Documentation Structure

All documentation must be placed under /docs using the following layout:

/docs
├── README.md
├── overview/
│   ├── project-overview.md
│   ├── architecture-overview.md
│   └── technology-stack.md
├── documentations/
├── features/
│   ├── feature-list.md
│   ├── feature-<feature-name>.md
│   └── feature-dependencies.md
├── workflows/
│   ├── workflow-summary.md
│   └── <workflow-name>-sequence.md
├── diagrams/
│   ├── system-context.mmd
│   ├── container-diagram.mmd
│   ├── component-diagram.mmd
│   ├── sequence-<use-case>.mmd
│   └── erd.mmd
└── decisions/
    ├── architectural-decisions.md
    └── assumptions.md
